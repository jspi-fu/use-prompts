# 五个关键问题法

```markdown
分析所有输入内容，生成5个关键问题。
回答这些问题后，应能完整捕捉所有内容的核心要点和中心意义。
```

---

## 课程阅读专用版

```markdown
审阅所有上传材料，生成 5 个抓住核心含义的问题。
重点关注：中心主题与定义、被强调的关键概念、概念之间的关系、提到的实际应用。
```

---

## 「最有趣发现」模板

```markdown
这些资料里最令人惊讶或最有趣的信息是什么？
请包含关键引文。
```

---

## 带方向性的扩展版「有趣发现」

```markdown
我打算写关于 [主题] 的内容。这些资料里与 [主题] 最令人惊讶的事实或想法是什么？

请包含关键引文。
重点关注 [具体方面]，不要涉及 [其他方面]。
```

---

## 竞赛式问答（音频概览）

```markdown
设计一个双主持人竞赛。
第一位主持人就 [主题] 向第二位提问，共10题，混合选择题和判断题。
答题者偶尔会答错，另一位主持人纠正并给出正确答案。
最后公布结果。
```

---

## 产品经理视角

```markdown
作为首席产品经理审阅内部文档。严格筛选可执行的洞察，忽略冗余内容。

以「决策备忘录」格式输出：
- **用户证据**：指向真实问题的直接引文
- **可行性检查**：提及的技术限制
- **盲点**：来源中缺失的内容

使用要点格式。
如果我的问题模糊，要求我澄清。
```

---

## 资深科学家角色

```markdown
扮演资深科学家的研究助理。语气严格客观、正式、精准。

假设具备 [领域] 高级知识，不解释标准术语。
重点关注方法论、数据完整性、矛盾证据。
优先考虑样本量、实验设计、统计显著性，而非泛泛结论。

以粗体分节：
- **关键发现**
- **方法论优缺点**
- **矛盾点**
```

---

## 中学教师角色

```markdown
扮演活力十足的中学老师。
将来源文档翻译成七年级学生能懂的语言。

每条回复结构：
1. 一句话总结（tl;dr）
2. 真实世界类比
3. 词汇表（3个难词的简单解释）

对于密集段落，转成判断题格式。
```

---

## 文献综述主题提炼

```markdown
从关于 [主题] 的文章中，找出 5–10 个最常出现的主题。

对每个主题提供：
- 自己的话简短定义
- 哪些文章提到（附引文）
- 一句说明处理方式（被争论、被假设、被证明）

以结构化表格呈现。
```

---

## 矛盾点挖掘

```markdown
从关于 [主题] 的文章中，找出重大矛盾或冲突发现。

对每处矛盾提供：
- 双方具体主张（附引文）
- 可能的分歧原因（方法、样本、语境）
- 什么证据能解决冲突
```
